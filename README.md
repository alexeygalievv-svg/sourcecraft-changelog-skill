# SourceCraft: генератор CHANGELOG

В репозитории описан **кастомный AI-навык** SourceCraft: файл  
`.sourcecraft/skills/changelog-generator.yaml`.

Навык берёт историю коммитов из выбранной ветки, отбрасывает merge-коммиты, группирует записи по типам **Conventional Commits** (`feat`, `fix`, `chore`, `docs`, `refactor`), добавляет эмодзи к секциям и выдаёт аккуратный Markdown для вставки в `CHANGELOG.md`.

## Как запустить в интерфейсе SourceCraft

1. Откройте репозиторий в SourceCraft и чат **Code Assistant**.
2. Вызовите навык **`changelog-generator`** (через список AI skills / slash-команды навыков — так же, как другие кастомные навыки из `.sourcecraft/skills/`).
3. Заполните параметры в UI и отправьте запуск.

## Входные параметры

| Параметр  | Тип     | Обязательный | По умолчанию | Описание |
|-----------|---------|--------------|---------------|----------|
| `version` | string | да           | —             | Номер релиза для заголовка (например `1.2.0`). |
| `branch`  | string | нет          | `main`        | Ветка, по которой читать `git log`. |
| `lang`    | ru / en | нет         | `ru`          | Язык заголовков и подписей в итоговом Markdown. |

Пример итогового файла для презентации демо — [CHANGELOG.md](./CHANGELOG.md).

## Problem solved

Ручная сборка списка изменений из `git log` для релиза обычно занимает **20–40 минут**: отсеивание merge-коммитов, группировка по типам, единый стиль формулировок и Markdown. Навык автоматизирует этот шаблон и стабилизирует формат между релизами, оставляя человеку только проверку и лёгкую правку формулировок.
